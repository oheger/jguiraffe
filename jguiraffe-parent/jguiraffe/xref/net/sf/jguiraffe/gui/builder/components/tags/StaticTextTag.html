24  /**
25   * <p>
26   * A tag handler class for creating a static text component.
27   * </p>
28   * <p>
29   * A static text is very similar to a label. The main difference is that it is
30   * treated as an input component and can thus be addressed by name and
31   * manipulated. Per default a static text component won't become part of the
32   * form that is constructed during the build operation. But for it a
33   * {@link net.sf.jguiraffe.gui.builder.components.ComponentHandler
34   * ComponentHandler} object is created that can be obtained from the
35   * <code>ComponentBuilderData</code> object. This handler can be used to
36   * manipulate the data of the static text.
37   * </p>
38   * <p>
39   * The properties of a static text are its text, its icon, and the alignment
40   * between the two. They can be read and written using objects of type
41   * {@link net.sf.jguiraffe.gui.builder.components.model.StaticTextData
42   * StaticTextData}. This can be done in the following way (assuming that
43   * <code>myStatic</code> is the name of a static text element and
44   * <code>builderData</code> is an instance of
45   * {@link net.sf.jguiraffe.gui.builder.components.ComponentBuilderData
46   * ComponentBuilderData}):
47   * </p>
48   *
49   * <pre>
50   * ComponentHandler handler = builderData.getComponentHandler(&quot;myStatic&quot;);
51   * StaticTextData data = (StaticTextData) handler.getData();
52   * // set a new text
53   * data.setText(&quot;Some new text&quot;);
54   * handler.setData(data);
55   * </pre>
56   * <p>
57   * Note that it is not sufficient to simply manipulate the
58   * <code>StaticTextData</code> object. Instead the altered object must be passed
59   * again to the <code>setData()</code> method of the component handler. The
60   * ability of being altered after the creation makes static text elements
61   * suitable for implementing dynamic labels. So simple labels, which won't be
62   * changed during the life time of a form, can be realized using the
63   * {@link LabelTag}, while this tag allows for instance labels whose text or
64   * icon is based on some calculations or change dynamically. The following
65   * attributes are supported by this tag handler class (in addition to the
66   * default attributes allowed for all input component tags):
67   * </p>
68   * <p>
69   * <table border="1">
70   * <tr>
71   * <th>Attribute</th>
72   * <th>Description</th>
73   * <th>Optional</th>
74   * </tr>
75   * <tr>
76   * <td valign="top"><code>text</code></td>
77   * <td>With this attribute the label's text can directly be set.</td>
78   * <td valign="top">yes</td>
79   * </tr>
80   * <tr>
81   * <td valign="top"><code>textres</code></td>
82   * <td>Defines the resource ID for the label's text. The real text is resolved
83   * using the current resource manager and the current locale.</td>
84   * <td valign="top">yes</td>
85   * </tr>
86   * <tr>
87   * <td valign="top"><code>resgrp</code></td>
88   * <td>Specifies the resource group of the label's text. If set, this resource
89   * group is used when resolving the label's text as defined by the
90   * <code>textres</code> attribute. If undefined, the form builder's default
91   * resource group will be used.</td>
92   * <td valign="top">yes</td>
93   * </tr>
94   * <tr>
95   * <td valign="top"><code>alignment</code></td>
96   * <td>Defines the relative position of the label's icon to its text. This can
97   * be one of the literal names of the
98   * {@link net.sf.jguiraffe.gui.builder.components.model.TextIconAlignment
99   * TextIconAlignment} class.</td>
100  * <td valign="top">yes</td>
101  * </tr>
102  * <tr>
103  * <td valign="top"><code>noField</code></td>
104  * <td>This boolean standard attribute determines whether this element is to be
105  * added to the form object. In contrast to most of the other input components
106  * it has a default value of <b>true</b>, meaning that static text elements are
107  * not part of the constructed form. If desired, this can be manually set to
108  * <b>false</b>.</td>
109  * <td valign="top">Yes</td>
110  * </tr>
111  * </table>
112  * </p>
113  * <p>
114  * This tag supports an icon, which can be defined by an {@link IconTag} in the
115  * tag's body. It is supported to define neither a text nor an icon. The static
116  * text's content may then be defined later.
117  * </p>
118  *
119  * @author Oliver Heger
120  * @version $Id: StaticTextTag.java 205 2012-01-29 18:29:57Z oheger $
121  */
122 public class StaticTextTag extends PushButtonTag
123 {
124     /**
125      * Creates a new instance of <code>StaticTextTag</code>.
126      */
127     public StaticTextTag()
128     {
129         super();
130         setNoField(true);
131     }
132 
133     /**
134      * Performs a validation for this tag. For a static text element no
135      * validation is needed because completely undefined tags are allowed, too.
136      *
137      * @throws FormBuilderException if the tag is incorrectly used
138      * @throws MissingAttributeException if a required attribute is missing
139      */
140     @Override
141     protected void validateTag() throws FormBuilderException,
142             MissingAttributeException
143     {
144     }
145 
146     /**
147      * Creates the element. This implementation will create a static text
148      * element using the specified component manager.
149      *
150      * @param manager the component manager
151      * @param create the create flag
152      * @return the component handler for the newly created element
153      * @throws FormBuilderException if an error occurs
154      */
155     @Override
156     protected ComponentHandler<?> createPushButton(ComponentManager manager,
157             boolean create) throws FormBuilderException
158     {
159         return manager.createStaticText(this, create);
160     }
161 }
