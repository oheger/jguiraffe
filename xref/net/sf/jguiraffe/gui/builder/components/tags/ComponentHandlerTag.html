32  /**
33   * <p>
34   * A tag handler class that allows constructing custom {@link ComponentHandler}
35   * objects.
36   * </p>
37   * <p>
38   * This tag can be placed in the body of a {@link FieldTag} to define the
39   * <code>ComponentHandler</code> that should be used by the input component to
40   * be constructed. The class of the new component handler must be specified in
41   * the <code>class</code> attribute. An instance of this class is created,
42   * initialized (which can be done by using other attributes representing
43   * properties of the newly created object), and finally passed to the enclosing
44   * <code>FieldTag</code> instance. This tag will then construct a suitable
45   * {@link net.sf.jguiraffe.gui.forms.FieldHandler FieldHandler} object and
46   * ensure that the represented component is added to the current
47   * {@link net.sf.jguiraffe.gui.forms.Form Form} object and to the enclosing GUI
48   * container element.
49   * </p>
50   * <p>
51   * With this tag handler class two common use cases can be addressed:
52   * <ul>
53   * <li>Components not supported by the form builder library (e.g. custom
54   * controls) can be added to both the resulting <code>Form</code> object and the
55   * generated GUI. For this purpose a suitable implementation of the
56   * <code>ComponentHandler</code> interface must be provided, which wraps the
57   * component to be added. The name of the implementation class must then be
58   * specified as the value of this tag's <code>class</code> attribute.</li>
59   * <li>Sometimes the default data provided by the supported GUI components is
60   * not sufficient. A typical example is a group of radio buttons: Each radio
61   * button has a boolean data object that tells whether this button is the
62   * selected one. Instead of storing a boolean value for each radio button that
63   * belongs to the group in the form's bean it is probably better to use a
64   * different storage format, e.g. storing only the index of the selected button
65   * as an int or using some kind of mapping to other values. This can be achieved
66   * by first defining the components that should be manipulated (in this example
67   * the radio buttons) in the usual way in the Jelly script, but setting their
68   * <code>noField</code> attribute to <b>true</b>. This ensures that they are not
69   * added as concrete data fields to the constructed <code>Form</code> object.
70   * Then an implementation of the {@link CompositeComponentHandler} interface
71   * must be created and specified in this tag's <code>class</code> attribute. In
72   * this tag's body an arbitrary number of {@link ReferenceTag} elements can be
73   * placed defining the components that should be wrapped by the new composite
74   * component handler. The components specified this way will be retrieved and
75   * added to the composite component handler. Then this composite handler will be
76   * added to the <code>Form</code> object as a complex data member. It can then
77   * perform an arbitrary transformation from the containing components' native
78   * data format to another format.</li>
79   * </p>
80   * <p>
81   * This tag of course supports all attributes already defined in its base class.
82   * If references are used to define components that are to be added to a
83   * composite component handler, these references are resolved at the very end of
84   * the builder process, thus ensuring that the referred components have already
85   * been created.
86   * </p>
87   *
88   * @see ReferenceTag
89   * @author Oliver Heger
90   * @version $Id: ComponentHandlerTag.java 205 2012-01-29 18:29:57Z oheger $
91   */
92  public class ComponentHandlerTag extends UseBeanBaseTag
93  {
94      /** Stores a collection with references to other components. */
95      private Collection<Reference> references;
96  
97      /**
98       * Adds a reference to another component to this tag. The reference will be
99       * resolved, and the corresponding component handler will be added to the
100      * newly created component handler (which must be a composite handler).
101      *
102      * @param ref the name of the referenced component
103      */
104     public void addComponentReference(String ref)
105     {
106         addReference(new ComponentReference(ref));
107     }
108 
109     /**
110      * Adds a reference to a component group to this tag. The component handlers
111      * of all components contained in this group will be fetched and added to
112      * the newly created component handler (which must be a composite handler).
113      *
114      * @param ref the name of the referenced group
115      */
116     public void addGroupReference(String ref)
117     {
118         addReference(new GroupReference(ref, getContext()));
119     }
120 
121     /**
122      * Adds a reference to the internal list.
123      *
124      * @param ref the new reference
125      */
126     protected void addReference(Reference ref)
127     {
128         if (references == null)
129         {
130             references = new LinkedList<Reference>();
131         }
132         references.add(ref);
133     }
134 
135     /**
136      * Passes the results of this tag to its target tag. This implementation
137      * checks whether the parent tag of this tag is of type {@link FieldTag}. If
138      * this is the case, the newly created bean instance is added as a component
139      * handler. If references to other components or groups are defined, the
140      * corresponding reference objects are scheduled for resolving (the new bean
141      * must then implement the <code>CompositeComponentHandler</code>
142      * interface).
143      *
144      * @param bean the newly created bean
145      * @return a flag whether the bean could be passed to the target
146      * @throws JellyTagException if an error occurs
147      */
148     @Override
149     protected boolean passResults(Object bean) throws JellyTagException
150     {
151         if (getParent() instanceof FieldTag)
152         {
153             ((FieldTag) getParent())
154                     .setComponentHandler((ComponentHandler<?>) bean);
155 
156             if (references != null)
157             {
158                 if (!(bean instanceof CompositeComponentHandler<?, ?>))
159                 {
160                     throw new JellyTagException(
161                             "A CompositeComponentHandler must be specified "
162                                     + "when defining references!");
163                 }
164                 ComponentBuilderData builderData = FormBaseTag
165                         .getBuilderData(getContext());
166                 for (Reference ref : references)
167                 {
168                     builderData.addCallBack(ref, bean);
169                 }
170             }
171             return true;
172         }
173 
174         return false;
175     }

177     /**
178      * Helper method for adding the {@code ComponentHandler} of a component to a
179      * {@code CompositeComponentHandler}. This method tries to obtain the
180      * component handler for the component with the specified name from the
181      * builder data object. Then it adds this handler to the complex handler.
182      * Note that it cannot be verified that the child {@code ComponentHandler}
183      * is actually of the expected data type. Therefore it is not possible to
184      * exclude that a {@code ClassCastException} might be thrown.
185      *
186      * @param <S> the type of the child {@code ComponentHandler}
187      * @param builderData the builder data object
188      * @param handler the {@code CompositeComponentHandler}
189      * @param compName the name of the component to add
190      * @throws FormBuilderException if the component cannot be found
191      */
192     protected static <S> void addComponentToCompositeHandler(
193             ComponentBuilderData builderData,
194             CompositeComponentHandler<?, S> handler, String compName)
195             throws FormBuilderException
196     {
197         ComponentHandler<S> ch = (ComponentHandler<S>) builderData
198                 .getComponentHandler(compName);
199         if (ch == null)
200         {
201             throw new FormBuilderException(
202                     "Cannot find component handler with name " + compName);
203         }
204         handler.addHandler(compName, ch);
205     }
206 
207     /**
208      * Helper method for adding the {@code ComponentHandler} objects for all
209      * components in the specified group to a {@code CompositeComponentHandler}.
210      * This method iterates over all components in the specified group and calls
211      * {@code #addComponentToCompositeHandler()} for each.
212      *
213      * @param builderData the builder data object
214      * @param handler the {@code CompositeComponentHandler}
215      * @param group the {@code ComponentGroup}
216      * @throws FormBuilderException if a component cannot be resolved
217      */
218     protected static void addGroupToCompositeHandler(
219             ComponentBuilderData builderData,
220             CompositeComponentHandler<?, ?> handler, ComponentGroup group)
221             throws FormBuilderException
222     {
223         for (String name : group.getComponentNames())
224         {
225             addComponentToCompositeHandler(builderData, handler, name);
226         }
227     }

229     /**
230      * An abstract base class for resolving references to other components.
231      * Instances of this class will be added as callbacks to the global
232      * <code>ComponentBuilderData</code> object. They are then invoked after all
233      * components have been created, which ensures that indeed all references
234      * can be resolved. Concrete sub classes deal with different types of
235      * references.
236      */
237     abstract static class Reference implements ComponentBuilderCallBack
238     {
239         /** Stores the name of the reference. */
240         private String referenceName;
241 
242         /**
243          * Creates a new instance of <code>Reference</code> and initializes it
244          * with the reference name.
245          *
246          * @param name the reference name
247          */
248         protected Reference(String name)
249         {
250             referenceName = name;
251         }
252 
253         /**
254          * Returns the reference name.
255          *
256          * @return the name of the reference
257          */
258         public String getReferenceName()
259         {
260             return referenceName;
261         }
262 
263         /**
264          * The callback method. Invokes <code>handleReference()</code> with the
265          * correct arguments.
266          *
267          * @param builderData the builder data object
268          * @param params the parameter object (a composite component handler)
269          * @throws FormBuilderException if an error occurs
270          */
271         public void callBack(ComponentBuilderData builderData, Object params)
272                 throws FormBuilderException
273         {
274             handleReference(builderData, (CompositeComponentHandler<?, ?>) params,
275                     getReferenceName());
276         }
277 
278         /**
279          * Processes the specified reference and adds all components it refers
280          * to to the given composite component handler.
281          *
282          * @param builderData the builder data object
283          * @param handler the handler to be initialized
284          * @param refName the name of this reference
285          * @throws FormBuilderException if an error occurs
286          */
287         protected abstract void handleReference(
288                 ComponentBuilderData builderData,
289                 CompositeComponentHandler<?, ?> handler, String refName)
290                 throws FormBuilderException;
291     }

293     /**
294      * A concrete <code>Reference</code> sub class that deals with (simple)
295      * component references.
296      */
297     static class ComponentReference extends Reference
298     {
299         /**
300          * Creates a new instance of <code>ComponentReference</code> and sets
301          * the reference name.
302          *
303          * @param refName the name of the reference
304          */
305         public ComponentReference(String refName)
306         {
307             super(refName);
308         }
309 
310         /**
311          * Processes this component reference.
312          *
313          * @param builderData the builder data object
314          * @param handler the handler to be initialized
315          * @param refName the name of this reference
316          * @throws FormBuilderException if an error occurs
317          */
318         @Override
319         protected void handleReference(ComponentBuilderData builderData,
320                 CompositeComponentHandler<?, ?> handler, String refName)
321                 throws FormBuilderException
322         {
323             addComponentToCompositeHandler(builderData, handler, refName);
324         }
325     }

327     /**
328      * A concrete <code>Reference</code> sub class that deals with group
329      * references. All components belonging to the specified group will be added
330      * to the composite handler.
331      */
332     static class GroupReference extends Reference
333     {
334         /** Stores a reference to the Jelly context. */
335         private JellyContext context;
336 
337         /**
338          * Creates a new instance of <code>GroupReference</code> and sets the
339          * reference name. The Jelly context must be passed in, too, because it
340          * is needed for resolving the group name.
341          *
342          * @param refName the name of the reference
343          * @param ctx the Jelly context
344          */
345         public GroupReference(String refName, JellyContext ctx)
346         {
347             super(refName);
348             context = ctx;
349         }
350 
351         /**
352          * Processes this group reference.
353          *
354          * @param builderData the builder data object
355          * @param handler the handler to be initialized
356          * @param refName the name of this reference
357          * @throws FormBuilderException if an error occurs
358          */
359         @Override
360         protected void handleReference(ComponentBuilderData builderData,
361                 CompositeComponentHandler<?, ?> handler, String refName)
362                 throws FormBuilderException
363         {
364             try
365             {
366                 ComponentGroup group = ComponentGroup.fromContext(context,
367                         refName);
368                 addGroupToCompositeHandler(builderData, handler, group);
369             }
370             catch (NoSuchElementException nex)
371             {
372                 throw new FormBuilderException("Cannot obtain group with name "
373                         + refName);
374             }
375         }
376     }
