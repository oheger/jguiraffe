31  /**
32   * <p>
33   * A tag handler class for defining columns of a table component.
34   * </p>
35   * <p>
36   * The tag represented by this handler class can be placed in the body of a
37   * {@link TableTag}. It defines the content of one of the table's columns. Its
38   * header text and its width can also be specified.
39   * </p>
40   * <p>
41   * In the most simple case a property from the beans that form the table's model
42   * is directly output. In this case only the {@code name} attribute needs to be
43   * set. The table implementation will then fetch this property from the model
44   * bean for the current row and display it without further modifications. It is
45   * recommended to set the {@code columnClass} attribute to the fully qualified
46   * name of the value class; then a suitable cell renderer can automatically be
47   * chosen (e.g. for rendering boolean values as checkboxes etc.).
48   * </p>
49   * <p>
50   * If additional formatting is required (e.g. for adapting number or date values
51   * to the locale of the current user), a <em>write transformer</em> can be
52   * specified. This transformer will be invoked before the value is displayed.
53   * </p>
54   * <p>
55   * To satisfy even more complex requirements a <code>&lt;colrenderer&gt;</code>
56   * tag (implemented by the {@link ColumnRendererTag} class can be placed in the
57   * body of this tag. This tag can be used to define an arbitrary component (even
58   * a container with an arbitrary number of child components) that will be used
59   * for painting the cells in this row. This is especially useful if a column in
60   * a table should contain a complete (sub) form.
61   * </p>
62   * <p>
63   * For editable columns similar variants are possible. In the simplest scenario
64   * only the property name (and eventually the column class) is specified. The
65   * table will then use an appropriate default cell editor for editing values in
66   * this column. If formatting and validating is involved, corresponding
67   * transformers and validators can be specified using nested tags. This happens
68   * in an analogous way as for standard input components. The most complex
69   * scenario is again to use a nested tag (an <code>&lt;coleditor&gt;</code> tag
70   * in this case) for defining a complete (sub) form as editor for this column.
71   * </p>
72   * <p>
73   * The following table lists all attributes supported by this tag:
74   * <table border="1">
75   * <tr>
76   * <th>Attribute</th>
77   * <th>Description</th>
78   * <th>Optional</th>
79   * </tr>
80   * <tr>
81   * <td valign="top">name</td>
82   * <td>This attribute defines the name of the property (of the beans that belong
83   * to the model) that is to be displayed in this column. The property will be
84   * read using reflection from the model beans. This attribute is mandatory even
85   * if specific forms are defined for both the column renderer and the column
86   * editor.</td>
87   * <td valign="top">No</td>
88   * </tr>
89   * <tr>
90   * <td valign="top">columnClass</td>
91   * <td>Here the class of the data to be displayed in this column can be
92   * specified. If no specific renderer and/or editor is defined for this column,
93   * the class will be used to determine a suitable default renderer or editor
94   * class. There are multiple ways of defining the column class using this
95   * attribute:
96   * <ul>
97   * <li>An instance of {@link ColumnClass} or the name of a constant defined in
98   * this class can be specified. In this case a logic type for the column is set.
99   * It will be interpreted and transformed according to the UI toolkit in use.
100  * This is the most portable way of defining the column class.</li>
101  * <li>A Java class or the fully-qualified name of a class can be specified.
102  * This class is passed to the underlying UI toolkit (which of course should be
103  * able to deal with it).</li>
104  * <li>If the column class is not specified, the most generic default renderer
105  * is used for this column.</li>
106  * </ul>
107  * </td>
108  * <td valign="top">Yes</td>
109  * </tr>
110  * <tr>
111  * <td valign="top">width</td>
112  * <td>Defines the absolute width of this table. The value specified here can be
113  * a {@link NumberWithUnit}.</td>
114  * <td valign="top">Yes</td>
115  * </tr>
116  * <tr>
117  * <td valign="top">percentWidth</td>
118  * <td>Using this attribute a relative width of this column in percent can be
119  * specified. If columns with a percent width are used, the algorithm to
120  * determine the widths of columns is as follows:
121  * <ul>
122  * <li>First columns with a fixed width are set. The remaining space is
123  * calculated (i.e. the width of the table minus the sum of the widths of the
124  * columns with a fixed width).</li>
125  * <li>For all columns that do not have a fixed width the percent values are
126  * determined: If a percent value is already set, it is used. For all other
127  * columns the percent width is set to a value so that the percent values sum up
128  * to 100 percent. (If this is not possible because the values specified sum up
129  * to a number greater than 100, an exception is thrown when creating the
130  * table.)</li>
131  * <li>The remaining space is distributed to the columns based on their percent
132  * value.</li>
133  * </ul>
134  * Using this algorithm it is possible to mix columns with fix and with relative
135  * sizes.</td>
136  * <td valign="top">Yes</td>
137  * </tr>
138  * <tr>
139  * <td valign="top">header</td>
140  * <td>With this attribute the header text for this column can directly be
141  * specified.</td>
142  * <td valign="top">Yes</td>
143  * </tr>
144  * <tr>
145  * <td valign="top">headerres</td>
146  * <td>Specifies the header text for this column using a resource ID.</td>
147  * <td valign="top">Yes</td>
148  * </tr>
149  * <tr>
150  * <td valign="top">resgrp</td>
151  * <td>Here are resource group for resolving the header text can be specified.
152  * This attribute is only evaluated if the <code>headerres</code> attribute is
153  * set. If no specific resource group is provided, the default group will be
154  * used.</td>
155  * <td valign="top">Yes</td>
156  * </tr>
157  * <tr>
158  * <td valign="top">editable</td>
159  * <td>This boolean attribute determines whether this column can be edited. If
160  * it is undefined, the value of the enclosing <code>&lt;table&gt;</code> tag's
161  * <code>editable</code> attribute is used.</td>
162  * <td valign="top">Yes</td>
163  * </tr>
164  * </table>
165  * </p>
166  * <p>
167  * In the body of this tag the <code>&lt;colrenderer&gt;</code> and
168  * <code>&lt;coleditor&gt;</code> tags can appear. Other content is not allowed.
169  * </p>
170  *
171  * @author Oliver Heger
172  * @version $Id: TableColumnTag.java 205 2012-01-29 18:29:57Z oheger $
173  */

445      * Performs processing before the tag's body is evaluated. This

449      * @throws FormBuilderException if an error occurs

451     @Override
452     protected void processBeforeBody() throws JellyTagException,
453             FormBuilderException
454     {
455         super.processBeforeBody();
456         tableTag = (TableTag) findAncestorWithClass(TableTag.class);
457         if (tableTag == null)
458         {
459             throw new FormBuilderException(
460                     "Column tag must be nested inside a table tag!");
461         }
462         if (strWidth != null)
463         {
464             width = new NumberWithUnit(strWidth);
465         }
466         initColumnClass();
467 
468         ComponentHandlerImpl handler = new ComponentHandlerImpl();
469         handler.setType(getDataClass());
470         setComponentHandler(handler);
471     }

531     /**
532      * Conditionally inserts the specified field handler into the given form.

534      * @param form the form

542         }

545     /**
546      * Initializes the class of this column. If a logic column class is defined,
547      * this one is used. Otherwise, it is checked whether a normal Java class is
548      * specified.
549      *
550      * @throws net.sf.jguiraffe.gui.builder.components.FormBuilderRuntimeException
551      *         if a conversion error of the data class occurs
552      */
553     private void initColumnClass()
554     {
555         if (columnClass instanceof ColumnClass)
556         {
557             logicClass = (ColumnClass) columnClass;
558         }
559 
560         else if (columnClass != null)
561         {
562             String clsName = columnClass.toString();
563             for (ColumnClass cc : ColumnClass.values())
564             {
565                 if (cc.name().equalsIgnoreCase(clsName))
566                 {
567                     logicClass = cc;
568                     break;
569                 }
570             }
571 
572             if (logicClass == null)
573             {
574                 dataClass = convertToClass(columnClass);
575             }
576         }
577     }

579     /**
580      * Helper method for obtaining a set with column names. Ensures that always
581      * a non-modifiable, non-null set is returned.
582      *
583      * @param s the source set
584      * @return the resulting set
585      */
586     private static Set<String> fetchSet(Set<String> s)
587     {
588         Set<String> result;
589         if (s != null)
590         {
591             result = Collections.unmodifiableSet(s);
592         }
593         else
594         {
595             result = Collections.emptySet();
596         }
597         return result;
